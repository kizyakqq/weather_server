***УСТАНОВКА И ЗАПУСК***

# 1. Клонируйте репозиторий или создайте проект
mkdir weather_server && cd weather_server

# 2. Создайте виртуальное окружение
python -m venv venv

# 3. Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 4. Установите зависимости
pip install -r requirements.txt

# 5. Запустите сервер
python script.py

***API Endpoints***

1. POST /weather/current (Получение текущей погоды по координатам)
2. POST /cities (Добавление города для отслеживания)
3. GET /cities (Получение списка отслеживаемых городов)
4. GET /weather/{city_name}/{time} (Получение погоды для города на указанное время):
    Возвращает погодные данные для указанного города и времени. Сервер ищет в БД ближайшую запись к запрошенному времени в пределах текущего дня.
    *Параметры пути:*
        city_name — Название города (например, Moscow)
        time — Время в формате ISO 8601 (например, 2026-02-16T12:00:00)
    *Параметры запроса (опционально):*
        params — Список запрашиваемых параметров. Возможные значения:
            temperature
            humidity
            wind_speed
            precipitation
            pressure
    Если параметр не указан, возвращаются все доступные данные.

Uvicorn 0.27.0 ASGI-сервер для запуска приложения

aiosqlite 0.19.0 Асинхронный драйвер SQLite
httpx 0.26.0 Асинхронный HTTP-клиент для запросов к API
Pydantic 2.5.3 Валидация данных и сериализация
